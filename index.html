Introduction

As BacklotCars continues to grow, both as a product and a company, one challenge we are faced with is learning how to refine the brand identity and apply it cohesively to each of our products. We created this styleguide to act as a central location where we store all the inventory of UI components, brand guidelines, brand assets, code snippets, developer guidelines and more. Anyone working on the BacklotCars products is encouraged to stay familiar with this styleguide and help ensure that it is kept up-to-date.

Goals

Cohesive design language

Refine our visual language to be simpler, more fun and more communicative.

Platform agnostic system

Develop a single, unified system that is transferrable across all platforms and devices.

Expedite the design process

By abstracting our base styles layer, now developers can churn out UIs with minimal design input.

Setup

Backlot Cars Next.js Front-end

Requirements

Node.js version 12.13.0 LTS, includes npm 6.12.0

After cloning use 

npm install
to install all dependencies.

Available Scripts

In the project directory, you can run:

1. 
npm run next:dev

Runs the app in the development mode.

Open http://localhost:3001 to view it in the browser.

The page will reload if you make edits.

2. 
npm run next:build

Builds the app for production to the 

.next
folder.

Your app is ready to be deployed!

See the section about deployment for more information.

3. 
npm run docz:dev

Runs the style guide in the development mode.

Open http://localhost:3000 to view it in the browser.

The page will reload if you make edits.

4. 
npm run docz:build

Builds the style guide for production to the 

.docz
folder.

The style guide is ready to be deployed!

5. 
npm run fix-code

Automatcially update components based on linting rules Read more.

6. 
npm run fix-styles

Automatically update styles based on linting rules Read more.
